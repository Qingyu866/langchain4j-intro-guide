核心概念

深入理解 LangChain4j 的核心概念，掌握框架的设计哲学和基础架构。

ChatModel

ChatModel 是与 LLM 交互的核心接口，支持多种模型提供商。

Java 
ChatLanguageModel model = OpenAiChatModel.builder()
    .apiKey(System.getenv("OPENAI_API_KEY"))
    .build();

String response = model.generate("Hello!");

通过统一的接口，轻松切换不同的 LLM 提供商。

AI Services

AI Services 是 LangChain4j 的核心特性，通过声明式接口大幅简化开发。

Java 
interface Assistant {
    String chat(String message);
}

Assistant assistant = AiServices.builder(Assistant.class)
    .chatLanguageModel(model)
    .build();

String response = assistant.chat("Hello!");

类似 Spring Data JPA 的体验，无需手动处理 prompt、token 管理。

ChatMemory

ChatMemory 用于管理对话历史，实现多轮对话。

  • • MessageWindowChatMemory - 保留最近 N 条消息
  • • TokenWindowChatMemory - 基于 token 数量管理
  • • ChatMemoryProvider - 支持数据库持久化

RAG

检索增强生成（RAG）结合检索和生成，提升 AI 回答的准确性。

RAG 工作流程

  1. 将文档切片并转换为 embedding
  2. 存储到向量数据库
  3. 用户提问时检索相关文档
  4. 将检索结果和问题一起发送给 LLM
